# Создание DAG для Airflow для анализа продаж видеоигр за 2011 год

## Описание проекта
Данный проект представляет собой создание DAG для Apache Airflow, который автоматизирует процесс регулярной выгрузки и аналитического исследования данных о продажах видеоигр по всему миру на основе датасета `vgsales.csv`. 

## Стек технологий
* **Python** (pandas, StringIO)
* **Apache Airflow 2.x** (Taskflow API: декораторы `@dag` и `@task`)

## Архитектура DAG
DAG построен по современной парадигме Taskflow API и состоит из **7 последовательно-параллельных задач (тасок)**:

1. `get_data` — загружает исходный набор данных по сети, трансформирует DataFrame в формат CSV-строки для корректной сериализации и передачи между тасками Airflow.
2. `get_top1_sales` — определяет самую продаваемую игру в мире за 2011 год.
3. `get_top_sales_genre_eu` — находит самый продаваемый жанр (или несколько) в Европе в 2011 году.
4. `get_top_sales_platform_na` — определяет платформы с наибольшим количеством миллионных тиражей в Северной Америке в 2011 году.
5. `get_top_sales_mean_publisher_jp` — находит издателей с наивысшими средними продажами в Японии в 2011 году.
6. `get_number_game_eu_more_jp` — рассчитывает количество игр, которые продавались в Европе лучше, чем в Японии в 2011 году.
7. `print_data` — финальный таск, который собирает результаты всех вычислений, получает текущую дату выполнения через `get_current_context()` и выводит итоговый отчет в логи Airflow.

## Как запустить
1. Поместите файл с кодом DAG в вашу рабочую директорию с дагами на сервере Airflow.
2. В веб-интерфейсе Airflow активируйте DAG `lesson_3_nikolaj-sofinskij-pwt6989`.
3. После успешного выполнения (Success) перейдите в логи таски `print_data`, чтобы посмотреть результаты анализа.

> ⚠️ **Примечание о доступности данных:** Исходные данные загружаются по внешней ссылке из облачного хранилища курса. В случае изменения путей к файлу или его недоступности в будущем, путь в функции `get_data()` потребуется обновить на актуальный локальный или сетевой адрес.
